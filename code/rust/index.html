开启Rust之旅

每每开始学习一门语言，我都会从官网的文档开始。因为，语言的文档是非常关键的，她告诉你什么是需要注意的，什么是重点，当然最重要的是她能做什么。

0x00 Rust，我为什么选择你

一门赋予每个人构建可靠且高效软件能力的语言。

高性能

Rust速度惊人且内存利用率极高。由于没有运行时和垃圾回收，它能够胜任对性能要求特别高的服务，可以在嵌入式设备上运行，还能轻松和其他语言集成。

  • 高性能如何来看待，我觉得还是要看在运算的过程中，快速的完成任务。

可靠性

Rust丰富的类型系统和所有权模型保证了内存安全和线程安全，让您在编译期就能够消除各种各样的错误。

  • 可靠性？就是你在使用Rust的时候bug少点，能及时发现你写的代码问题，当然规避一些错误。

生产力

Rust 拥有出色的文档、友好的编译器和清晰的错误提示信息， 还集成了一流的工具 —— 包管理器和构建工具， 智能地自动补全和类型检验的多编辑器支持， 以及自动格式化代码等等。

  • 生产力我理解的就是文档全面，有什么困难都能在文档中找到文档，构建工具方便。

0x01 环境准备

一切从文档开始,中文的文档也好，英文的文档也好，主要得把该将的说明白。

安装

我一直都喜欢用brew来安装，但是发现Rust安装后会有些问题，所以推荐使用官方的方式，就是在命令行安装。因为官方安装后就可以直接使用cargo了。

curl https://sh.rustup.rs -sSf | sh

安装过程比较快，是我最近安装dotnet-SDK和OpenJDK中最快的，或许本身语言比较年轻吧。

安装成功后，会显示以下信息：

Rust is installed now. Great!

当前终端Rust环境不会直接起效，所以需要手工进行处理：

source $HOME/.cargo/env

当然，你也可以重启终端。在配置文件~/.bash_profile下，发现了在最后行加入了以下的信息：

export PATH="$HOME/.cargo/bin:$PATH"

我比较喜欢的是，文档中居然把如何自我更新和删除放到了一块，就好比进入去买鱼，去了酱油店一样，说：“哦，不好意思我走错了！”。让你有充分的选择，喜欢吗？哦，不喜欢，那欢迎下次光临。

更新

rustup update

删除

rustp self remove

验证安装

rustc --version

0x02 Hello,World!

我感觉和世界问好已经成为了一种标准，还是习惯，大家都这样写了。

创建项目目录

mkdir project

创建文件main.rs,添加以下内容：

fn main() {
    println!("Hello, world!");
}

文件名称可以随意，不影响，不像go的主文件必须是main.go。

编译

rustc main.rs

运行

./main

结果输出：

Hello, world!

看似简单的一步完成了，如果没有编程经验的人会非常兴奋，毕竟入门了；对于有编程经验的人来说，开始入坑。

在Rust的安装，编译的过程都非常顺利，比较害怕的是弄个环境都可以搞一天的。

0x03 Hello,Cargo!

Cargo是Rust语言的包管理工具，非常棒的工具，我觉得go可以学习一下。

创建项目

cargo new hello_cargo

编译项目

cargo build

运行项目

cargo run

Cargo.toml

包管理配置文件，不是json不是xml。

[package]
name = "hello_cargo"
version = "0.1.0"
authors = ["Your Name <you@example.com>"]
edition = "2018"

[dependencies]

和make的配置文件相似，非常有亲切感。

0x04 猜谜游戏

这是hello的升级版，项目还是得使用cargo，毕竟需要些第三方的包呀，不能重复造轮子。

创建一个新项目guessing_game

cargo new guessing_game

创建后，加入以下内容：

use std::io;

fn main() {
    println!("Guess the number!");

    println!("Please input your guess.");

    let mut guess = String::new();

    io::stdin().read_line(&mut guess)
        .expect("Failed to read line");

    println!("You guessed: {}", guess);
}

代码包括了非常多的信息，我们来一行一行的解释：

引入io包
use std::io;

我们需要io包中的输入/输出功能。

存储信息
let mut guess = String::new();
引入第三方包
[dependencies]
rand = "0.5.5"

第三方的包版本控制需要手动进行修改，使用cargo update仅仅显示当前的最新版本，而不会进行替换。如果需要最新版本，需要手工进行配置Cargo.toml文件。

[dependencies]
rand = "0.6.0"

需不需要新功能，还是你说了算！

这里顺便提一下，我使用了vs code的rust插件后，使用cargo build的时候会出现以下问题：

Blocking waiting for file lock on package cache

两个线程在打架了，而vs code下的插件不能显示具体的下载过程，所以只能删除了，等出正是版本再说了。

解决该问题，可以删除.cargo文件夹下的.package_cache文件。
